Projects

Travelling Senator Problem

This was a project that I worked on as part of a team of four people. The problem was a variation of the famous traveling salesman problem. The task was to design an algorithm that, given a strongly-connected graph where each edge had a weight from 0 to 100 and each node was either colored blue or red, would return an approximation of the lowest cost path that would visit all the nodes once. However, the path could not visit more than 3 nodes of the same color in a row. My team had several approaches to this problem, and I was in charge of implementing a greedy algorithm that would pick the cheapest edge as the next edge to traverse in the path, without violating the constraint of visiting 3 nodes of the same color in a row. I implemented this algorithm in Java.

Pacman AI solver

In this two-member team project, my partner and I took several approaches to develop an AI for the classic Pacman game. We implemented various search algorithms to get Pacman to eat all of the food on the game table as quickly as possible, such as breadth-first search, depth-first search, uniform cost search, greedy search and A* search. Of all of these, A* search was by far the most effective. In a slightly different version of the game where there are ghosts in addition to food in the maze, we used the minimax and expectimax algorithms to model the way the ghosts were behaving, and thereby gave Pacman the best chance of beating the ghosts and winning the game.

Census Data App

This was an independent project I recently undertook. I developed the app over the course of a week. I made the app using the Xcode and wrote it primarily in Objective-C, though I used the Storyboard feature to specify the UI of the app. The app retrieves census data using the API provided by USA Today. You can use the app to retrieve census information about any town in the U.S. For instance, if I wanted to get the population mass or population density of Berkeley, California, I could enter that into the app, and it would retrieve the data for me. It also sorted the data by state, so I could also easily look up the population of Berkeley, Illinois if I so chose. I am also working on a version for Android using the Android SDK.

Autostereogram Generator

I wrote the code for this project in C. It is fed two images, one corresponding to a left hand side image, and the other corresponding to a right hand side image. I programmed the code to produce a depth map that maps the displacements between the left and the right images. This depth map can be used in the creation of 3D vision by other programs.In the second part of the project, I designed a storage space-saving feature whereby the depth map can be stored in memory more efficiently using a tree-structure. Sections of the depth map that had the same value assigned to them were stored as a single value in the leaf of a tree, thereby saving space.

Huffman Encoding

In this four-member team project, my partners and I used wrote code in Java to compress files. Think about how you compress files on your computer using WinZip or some other compression software -- this project implemented a very simple version of the same thing using the Huffman Encoding algorithm. The frequency of each character was counted and based on that, every character was encoded to a binary string. The characters that appeared more frequently were given shorter strings, and vice-versa, thereby saving space. Our code could also, given an encoded file and it's mapping, decode the file into the original, and can also compress entire directories.
